# DeployFlow Auto-Deployer (C-based CLI Tool)

เครื่องมือช่วยอัปโหลดโค้ดเว็บไซต์และโปรเจกต์ต่าง ๆ ขึ้น GitHub และ Deploy ขึ้น Server/Hosting อย่างสะดวกและรวดเร็วใน **คลิกเดียว (One-Click Deployer)** เขียนด้วยภาษา C น้ำหนักเบาและรันได้ทันทีโดยไม่ต้องลงทรัพยากรเพิ่มเติม

---

## คุณสมบัติเด่น (Features)

1. **ระบบตรวจจับอัจฉริยะ (Smart Auto-Detection)**:
   * หากตั้งค่าในคำสั่งเป็น `auto` โปรแกรมจะค้นหาโครงสร้างไฟล์ในเครื่องอัตโนมัติ:
     * **ระบบ Build**: หากพบ `package.json` จะรัน `npm run build`, หากพบ `Cargo.toml` จะรัน `cargo build --release` ฯลฯ
     * **ระบบ Deploy**: หากพบ `firebase.json` จะดีพลอยผ่าน `firebase deploy`, หากพบ `vercel.json` จะใช้ `vercel --prod` ฯลฯ
2. **ระบบสลับตัวเลือกสภาพแวดล้อม (Multi-Environment)**:
   * รองรับการใช้ไฟล์คอนฟิกหลายรูปแบบ เช่น รัน `.\deployer.exe staging` เพื่อดึงไฟล์คอนฟิก `deploy.staging.conf` มาใช้ดีพลอยตามจุดประสงค์ที่ต่างกัน
3. **ระบบตรวจสอบความพร้อมก่อนรัน (Pre-flight Checks)**:
   * ตรวจเช็คอินเทอร์เน็ตและการติดตั้งเครื่องมือที่จำเป็น (เช่น `git`, `npm`, `firebase`) หากไม่พร้อมระบบจะเตือนและหยุดรันงานทันทีเพื่อป้องกันความเสียหาย
4. **ระบบแจ้งเตือนผ่านหน้าต่าง Windows (Toast Notification)**:
   * มีหน้าต่าง Balloon Pop-up แจ้งผลลัพธ์ความสำเร็จหรือข้อผิดพลาดที่มุมขวาล่างของจอภาพทันที
5. **ระบบเก็บบันทึกประวัติ (Local Logging)**:
   * บันทึกความคืบหน้า วันเวลาที่สำเร็จ หรือจุดที่เกิดข้อผิดพลาดในการดีพลอยแต่ละครั้งลงในไฟล์ `deploy.log` ในโฟลเดอร์โปรเจกต์

---

## วิธีนำไปใช้งานในโปรเจกต์ของคุณ

1. คัดลอกไฟล์สองไฟล์นี้ไปวางไว้ที่โฟลเดอร์หลักของโครงการ (Root Directory):
   * **`deployer.exe`**
   * **`deploy.conf`**
2. เปิดไฟล์ `deploy.conf` เพื่อตั้งค่าโปรเจกต์ตามที่ต้องการ (ดูหัวข้อการตั้งค่าด้านล่าง)
3. เริ่มดีพลอยด้วยการ **ดับเบิลคลิกไฟล์ `deployer.exe`** หรือพิมพ์คำสั่งใน Terminal:
   ```powershell
   .\deployer.exe
   ```

---

## การตั้งค่าไฟล์คอนฟิก (`deploy.conf`)

ตัวอย่างไฟล์คอนฟิกพื้นฐาน:
```ini
# ตั้งค่าเป็น 1 เพื่อเปิดใช้ Git commit & push อัตโนมัติ (ใส่ 0 เพื่อปิด)
git_enabled = 1

# ชื่อสาขาปลายทางของ Git
git_branch = main

# ข้อความ Commit (หากปล่อยว่างไว้ระบบจะสุ่มชื่อตามวันและเวลาปัจจุบัน)
commit_message = Auto deploy: updated project files

# คำสั่ง Build (ใส่ 'auto' หรือใส่คำสั่งกำหนดเอง หรือทิ้งว่างหากไม่มี)
build_command = auto

# คำสั่ง Deploy (ใส่ 'auto' หรือใส่คำสั่งกำหนดเอง หรือทิ้งว่างหากไม่มี)
deploy_command = auto
```

---

## การรันแยกสภาพแวดล้อม (Environment)

หากคุณทำหลายสเตจ เช่น ต้องการทดสอบดีพลอยขึ้นระบบ **Staging**:
1. สร้างไฟล์ชื่อ `deploy.staging.conf` ขึ้นมาในโปรเจกต์
2. รันแอปพลิเคชันโดยต่อท้ายคีย์เวิร์ด:
   ```powershell
   .\deployer.exe staging
   ```
   *(ระบบจะข้ามไปดึงคอนฟิกในไฟล์ staging มาทำงานแทนไฟล์หลักทันที)*

## วิธีแก้ไขซอร์สโค้ดและคอมไพล์ใหม่

หากคุณเป็นผู้พัฒนาและทำการปรับแต่งโค้ดภาษา C ในไฟล์ `deployer.c`:
* ให้รันไฟล์ **`compile.bat`**
* ระบบจะทำการดาวน์โหลดคอมไพเลอร์ TCC ขนาดเล็กแบบพกพามาให้ และทำการคอมไพล์ซอร์สโค้ดออกเป็น `deployer.exe` ตัวใหม่ล่าสุดโดยอัตโนมัติ
